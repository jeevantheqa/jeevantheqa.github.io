A2 (Automation of Automation) Test Automation

Abstract

This paper attempts to explore a Test Automation Framework which helps non-programmers to seamlessly author automation scripts through a domain specific language based on English . These scripts can be automatically converted to executable Selenium Webdriver code and this design flow makes it easy to understand and makes maintance simpler. .

Problem Statement

Test Automation involves developing executable test scripts which significantly reduces the manual effort involved and plays a greater role in software testing industry . The key problems in existing tools of test automation are : .

  • Development of automation scripts is essentially backed behind strong programming skills.
  • The scripts are not always readable unless its been wrapped over Cucumber framework.
  • As the industry is moving towards Agile and DevOps , maintance of the code from frequent requirement changes is an intensive task .

Solution

The A2 Test Automation Framework enables the non-programmer testers to design automation test scripts using domain specific language (DSL). The DSL follows an English-like syntax making way for the testers to design and maintain the scripts with ease . Now the A2 Framework converts the English-like statements into Java Code using Selenium Webdriver API’s . The DSL grammar can be developed using Xtest and Xbase frameworks using the Gherkin syntax .The Java code generation can be implemented using Xtend programming language . .
